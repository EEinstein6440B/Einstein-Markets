================================================================================
                            BRIEFING.JSON SCHEMA
================================================================================

marketMood: string               Current market sentiment/color
generatedAt: ISO8601 timestamp   When briefing was generated

For each ticker:
  signal: "strong_buy"|"buy"|"neutral"|"caution"|"bearish"
  signalLabel: string            "Strong Buy Signal", "Neutral", etc.
  commentary: string             Analysis explanation
  momentum: "up"|"down"|"flat"   Price momentum
  timestamp: string              When analysis was generated
  consensus: {                   Analyst ratings distribution
    strongBuy: number,
    buy: number,
    hold: number,
    sell: number,
    strongSell: number
  }

Generated by scripts/generate-briefing.js and committed by GitHub Actions.

================================================================================
                          API INTEGRATION DETAILS
================================================================================

FINNHUB API:
- Base URL: https://finnhub.io/api/v1
- Endpoints used:
  /quote              (stock prices)
  /company-news       (company news)
  /stock/recommendation (analyst consensus)
- Rate limit: 60 requests/minute (free tier)
- Key storage: localStorage key 'fh_key'
- Used by: index.html (client) and generate-briefing.js (server)

COINGECKO API:
- Base URL: https://api.coingecko.com/api/v3
- Endpoints used:
  /simple/price       (BTC, ETH prices)
- Rate limit: Unlimited (community tier)
- No API key required
- Used by: index.html (client) and generate-briefing.js (server)

LOCALSTTORAGE:
- fh_key: Finnhub API key (user-entered in Settings)

================================================================================
                        GITHUB ACTIONS WORKFLOW
================================================================================

Trigger: Scheduled on cron expressions
  - 0 11 * * 1-5  (6 AM ET = 11 AM UTC, Monday-Friday)
  - 0 21 * * 1-5  (4 PM ET = 9 PM UTC, Monday-Friday)

Steps:
  1. Checkout repository
  2. Setup Node.js 18
  3. Install dependencies (if needed)
  4. Run scripts/generate-briefing.js with FINNHUB_API_KEY
  5. Commit data/briefing.json with timestamp

Requirements:
  - FINNHUB_API_KEY secret must be set in GitHub repo settings
  - Repository must have write permissions enabled

Output:
  - data/briefing.json updated with current analysis
  - Commit message: "chore: update market briefing YYYY-MM-DD HH:MM:SS UTC"
